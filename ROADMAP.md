# Roadmap

## Objetivo

Construir una web personal que funcione como:

- landing de marca profesional
- panel de control tipo "Mission Control"
- portfolio técnico
- CV descargable
- base para páginas técnicas profundas por proyecto

La prioridad no es solo que se vea bien.
La prioridad es que parezca una plataforma real y que explique valor rápido.

Fuente de verdad actual para copy, experiencia, proyectos, stack y enlaces: [CONTENT.md](/Users/sergioballesteros/Documents/cv-sergioballesteros/CONTENT.md).

## Principios

- No hacer el típico portfolio de `Hero / About / Skills / Projects / Contact`.
- No inventar métricas.
- No sobrecargar con animaciones.
- No parecer una demo de IA.
- Mantener claridad para recruiters y profundidad para perfiles técnicos.
- Reutilizar la misma identidad visual en web, PDF y futuras páginas de proyecto.

## Fase 1 - Base visual

### Entregables

- Home principal con identidad "Mission Control"
- Hero con retrato real
- Estado general del perfil
- Bloques de sistemas / proyectos
- Links a GitHub y LinkedIn
- CTA de descarga de CV

### Criterio de éxito

- Se entiende en menos de 20 segundos.
- No parece una plantilla.
- La foto suma confianza y no rompe el diseño.
- En móvil se mantiene legible.

## Fase 2 - Sistema de contenido

### Entregables

- inventario real de proyectos
- inventario real de experiencia
- lista de tecnologías
- lista de logros verificables
- frase de posicionamiento principal

### Criterio de éxito

- Cada frase fuerte tiene respaldo.
- Se puede reutilizar en home, CV y LinkedIn.
- El texto suena a backend / producto / datos / IA, no a marketing vacío.

## Fase 3 - Páginas de proyecto

### Entregables

- página para OCM Brain
- página para Energy Simulator Platform
- página para OCR + LLM
- página para Automated Import Pipelines
- página para Microservices Architecture

### Contenido por proyecto

- objetivo
- stack
- arquitectura
- decisiones técnicas
- reto resuelto
- resultado
- capturas o diagramas

### Criterio de éxito

- Un recruiter entiende el contexto rápido.
- Un técnico puede profundizar sin salir del sitio.
- Cada proyecto parece un sistema vivo, no una tarjeta de portfolio.

## Fase 4 - Documentación técnica

### Entregables

- MDX para proyectos
- diagramas Mermaid
- snippets de arquitectura
- notas de decisiones técnicas

### Criterio de éxito

- El contenido técnico no está escondido.
- Se puede mantener sin tocar el diseño principal.
- Es fácil añadir o editar proyectos.

## Fase 5 - CV / ATS

### Entregables

- versión imprimible limpia
- PDF descargable
- estructura semántica
- orden optimizado para ATS

### Criterio de éxito

- el PDF no pierde legibilidad
- el contenido clave sigue siendo indexable
- el diseño sigue siendo serio y profesional

## Fase 6 - Pulido final

### Entregables

- animaciones mínimas y útiles
- estados hover/focus limpios
- responsive fino
- detalles de iconografía
- microcopy final

### Criterio de éxito

- no hay ruido visual
- no hay saltos raros en móvil
- no hay texto inventado ni relleno
- la experiencia se siente premium

## Fase 7 - Publicación

### Entregables

- despliegue estable
- dominio propio si se quiere
- revisión final de performance
- revisión final de accesibilidad básica

### Criterio de éxito

- la web se puede compartir con un recruiter hoy
- la web también aguanta a un perfil técnico
- cambiar contenido no rompe el diseño

## Backlog

Estas ideas quedan para después, solo si aportan de verdad:

- terminal interactiva
- modo oscuro opcional
- sistema de search interno
- feed de deploys reales
- timeline de actividad
- mini observabilidad del portfolio

## Reglas de Prioridad

1. Primero identidad y claridad.
2. Después contenido real.
3. Después profundidad técnica.
4. Después efectos visuales.
5. Nunca al revés.

## Próximo Paso Recomendado

Definir el contenido real de:

- experiencia
- proyectos
- stack
- logros
- enlaces

Con eso se puede cerrar la home definitiva y luego abrir las páginas técnicas.
